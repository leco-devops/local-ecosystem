Service hubs

Each hub is a single-service operations page for the local ecosystem: what the service does, how it fits the platform, live Docker status, credentials, connection strings, and every *.lh URL you can open from the browser.

Use hubs when you need passwords, JDBC/Redis URLs, MinIO keys, or CF-local adapter endpoints without digging through compose files. Status and metrics refresh with the dashboard toolbar.

UI access (local dev)

Store login credentials for MinIO console, Adminer, n8n, and Open WebUI (control panels with sign-in only). Auto-login opens a short-lived magic link (~60s). Reset & apply restores compose defaults or wipes app volumes when needed. Requires the control token on the Control tab when DASHBOARD_CONTROL_TOKEN is set.
