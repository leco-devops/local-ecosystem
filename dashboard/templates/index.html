CPU (%)

Cyan: Docker ≈% of host vCPUs. Magenta: CPU from /proc (host mount on Linux, or the container’s /proc on Docker Desktop = Linux VM). Orange + right axis: temperature from /sys.

RAM (% · GB)

Y-axis is percent; hover tooltips and the summary line also show GB (Docker Σ vs MemTotal or Σ limits; magenta line ≈ avail or used vs MemTotal). Cyan: Docker Σ ÷ kernel MemTotal. Magenta: MemAvailable ÷ MemTotal (headroom) — not the same quantity as cyan. Without /proc, second line is Docker ÷ Σ cgroup limits.

Network (Mb/s)

Cyan: Docker aggregate RX+TX Mb/s. Magenta: host/VM from /proc/net/dev when readable.

IOPS & block I/O

Left: IOPS (often 0 on Docker Desktop). Right: Docker block throughput Mb/s

Host CPU temperature

Linux: max of /sys/class/thermal/*/temp (optional host mount DASHBOARD_HOST_SYS in dashboard.sh). macOS + Docker Desktop: optional brew install osx-cpu-temp. Starting the dashboard from ai-stack/services/dashboard.sh installs a LaunchAgent that runs ai-stack/scripts/macos-write-cpu-temp.sh every 30s; stopping/removing the dashboard removes it. On some Apple Silicon setups osx-cpu-temp reports 0°C; the writer then uses sudo powermetrics and maps thermal pressure to a proxy °C, plus load average so the chart moves under CPU load even when pressure stays Nominal. For unattended runs, passwordless sudo for /usr/bin/powermetrics may be needed. Deploy from the Mac so ~/.local-eco-host-metrics mounts into the container.

No temperature samples yet — on Mac run the host temp script; on Linux ensure /sys thermal zones exist or mount host /sys.

    Control

    Cards are grouped: AI-stack bulk, AI services, Infra add-ons (MySQL, Adminer, Redis Commander, cache lab, …), then Cloudflare local. Registered leco compose apps live under the Hosted apps tab. Each card is a stack or service — click an action. Destructive ops may wipe volumes; read the JSON response.

    All AI-stack services Uses bulk_ecosystem in ai-stack/core.sh. Bulk stop, restart, redeploy, pause, remove, reset, and recreate skip this dashboard and, by default, Traefik and Postgres so routing and the shared DB stay up. Override skips with ECOSYSTEM_BULK_PLATFORM_SKIP (space-separated names). To stop or redeploy Traefik, Postgres, or the dashboard itself, use the shell — see DEPLOYMENT.md (section Core infra — shell).
    Infra add-ons (compose) infra/docker-compose.yml — MySQL, Redis, Mailpit, Adminer, Redis Commander, Telegram gateway, cache lab. Same control token. Remove/reset can delete volumes.
